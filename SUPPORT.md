# 💬 LoveOS — Support Guide

Thank you for your interest in LoveOS.  
This document explains how to get help, ask questions, and interact with the project in a way that respects its philosophy and structure.

---

# 1. Before Asking for Help

Please review the following documents first:

- `README.md` — Public overview  
- `README-DEVELOPER.md` — Developer guidance  
- `ARCHITECTURE.md` — System architecture  
- `CONTRIBUTING.md` — How to contribute  
- `SECURITY.md` — Security policy  

Most questions are answered in these files.

---

# 2. Asking Questions

If you still need help, follow this ritual:

### **1. Ground**
Identify the exact problem or question.

### **2. Clarify**
Include:

- What you were trying to do  
- What you expected to happen  
- What actually happened  
- Any error messages  

### **3. Align**
Make sure your question respects:

- Sovereignty  
- Identity  
- Modularity  
- Ritual structure  

### **4. Submit**
Open a GitHub issue with the label:

```
question
```

---

# 3. Getting Help With Development

If you are contributing code:

- Open an issue before starting major work  
- Discuss design and architecture  
- Ensure your work aligns with LoveOS philosophy  
- Document everything clearly  

---

# 4. Reporting Bugs

Use the ritual:

1. **Ground** — Describe the bug  
2. **Reproduce** — Provide steps  
3. **Impact** — Explain what it affects  
4. **Align** — Suggest a fix if possible  
5. **Submit** — Open an issue with the label:

```
bug
```

---

# 5. Getting Help With Security Issues

Do **not** open a public issue.  
Follow the instructions in `SECURITY.md`.
