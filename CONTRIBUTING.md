# 🤝 Contributing to LoveOS

Thank you for your interest in contributing to LoveOS.  
This project is more than software — it is a sovereign operating system built on identity, ritual, and legacy. Contributions must respect the philosophy and architecture of the system.

---

# 1. Core Principles for Contributors

All contributions must align with:

- **Sovereignty** — No hidden behavior, no silent overrides  
- **Modularity** — Engines and modules, not monoliths  
- **Identity** — Branding, sigils, and philosophy respected  
- **Ritual** — Major actions map to rituals or operator workflows  
- **Clarity** — Code must be readable, intentional, and documented  

---

# 2. How to Contribute

### Step 1 — Fork the Repository  
Create your own fork of the LoveOS repo.

### Step 2 — Create a Feature Branch  
Use descriptive names:

```
feature/ritual-engine-update
fix/sovereignty-check
docs/add-architecture-diagram
```

### Step 3 — Make Your Changes  
Follow the development standards:

- Modular  
- Operator-proof  
- Identity-driven  
- Ritual-based  
- Audit-friendly  

### Step 4 — Write Documentation  
Every new module, engine, or ritual must include:

- Purpose  
- Inputs  
- Outputs  
- Ritual or workflow mapping  
- Identity considerations  

### Step 5 — Submit a Pull Request  
Your PR must include:

- What you changed  
- Why you changed it  
- How it aligns with LoveOS philosophy  
- Any new rituals or workflows introduced  

---

# 3. Code Style

- Use clear, intentional naming  
- Keep functions small and modular  
- Avoid side effects  
- Include comments for all rituals, engines, and identity logic  
- Follow the repository map structure  

---

# 4. Ritual Mapping

Every major contribution must map to a ritual:

- Initialization  
- Grounding  
- Sovereignty  
- Identity  
- Integration  

If your contribution does not map to a ritual, it may not belong in LoveOS.

---

# 5. Communication

Before submitting large changes, open an issue to discuss:

- Design  
- Architecture  
- Ritual alignment  
- Identity impact  

---

# 6. Philosophy Alignment

LoveOS is not a generic open-source project.  
It is a **sovereign system** with a **specific identity** and **spiritual architecture**.

Contributors must respect:

- The sigils  
- The branding  
- The emotional logic  
- The sovereignty model  
- The legacy intention
